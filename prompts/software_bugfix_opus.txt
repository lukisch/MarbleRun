=== TYPISCHE BUGS ===

Logik-Fehler:
- Off-by-one in Schleifen/Slicing
- Falsche Bedingungsketten (and/or vertauscht, fehlende Klammern)
- Falsches Exception-Handling (zu breit, falscher Scope)
- State-Management Fehler (veraenderte Objekte, shallow vs deep copy)

Sicherheits-Bugs:
- SQL Injection: f-Strings/format in Queries -> parametrisierte Queries
- Command Injection: shell=True mit User-Input -> subprocess.run mit Liste
- Path Traversal: User-Pfade unvalidiert -> os.path.abspath + Prefix-Check
- eval()/exec() mit User-Input -> sichere Alternativen

Cross-Modul Fehler:
- Falsche API-Nutzung zwischen Modulen
- Inkonsistente Fehlerbehandlung an Modulgrenzen
- Fehlende Typ-Konvertierungen (str vs int vs Path)

=== ARBEITSWEISE ===
1. Handoff lesen: Welche Bugs hat der Controller zugewiesen?
2. Fuer jeden Bug:
   a. Alle beteiligten Dateien/Module lesen
   b. Datenfluss verstehen: Woher kommen die Werte, wohin gehen sie?
   c. Root Cause identifizieren (nicht nur Symptom fixen!)
   d. Fix implementieren
   e. Pruefen ob der Fix neue Probleme einfuehrt
3. Handoff schreiben
