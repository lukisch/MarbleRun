=== AUDIT-CHECKLISTE PRO PROJEKT ===

1. Crash-Risiken:
   - [ ] Alle except-Bloecke spezifisch?
   - [ ] Alle File-Zugriffe mit with-Statement?
   - [ ] Alle None-Rueckgaben geprueft vor Zugriff?
   - [ ] Alle os.path / pathlib Operationen sicher?
   - [ ] Encoding ueberall explizit (utf-8)?

2. Logik:
   - [ ] Schleifen-Grenzen korrekt (off-by-one)?
   - [ ] Default-Werte sinnvoll?
   - [ ] Edge Cases behandelt (leere Listen, leere Strings, 0)?
   - [ ] Boolean-Logik korrekt (De Morgan, Kurzschluss)?

3. Sicherheit:
   - [ ] Keine SQL Injection (parametrisierte Queries)?
   - [ ] Kein shell=True mit User-Input?
   - [ ] Keine hardcoded Secrets?
   - [ ] Kein eval()/exec() mit externem Input?
   - [ ] Pfad-Validierung bei User-Eingaben?

4. Robustheit:
   - [ ] Netzwerk-Operationen mit Timeout?
   - [ ] Datei-Operationen mit Fehlerbehandlung?
   - [ ] Ressourcen werden freigegeben (close, cleanup)?
   - [ ] Grosse Datenmengen: Streaming statt alles in RAM?

=== ARBEITSWEISE ===
1. Handoff lesen: Welches Projekt ist zugewiesen?
2. KOMPLETT einlesen: Alle .py/.js/.ts Dateien

4. Alle gefundenen Bugs priorisieren (A > B > C > D)
